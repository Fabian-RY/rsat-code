RSAT - get-orthologs-compara manual

DESCRIPTION

Returns orthologues, plus optionally paralogues and homoeologues, for a set of genes in one or more organisms. Relies on primary data from Ensembl Compara.

AUTHORS

Bruno Contreras Moreira
Jacques van Helden

INPUT FORMAT

Query genes can be directly entered or uploaded from an input file (option -i). The first word of each row of such a file is handled as a gene. Any additional information on the same row is ignored.

HOMOLOGY TYPES

Supported homology types and subtypes:

  • orth
      Returns only orthologous genes, including many2many, one2many & one2one.

  • para
      Returns only paralogues, including within_species_paralog, other_paralog & gene_split.

  • hom
      Returns only homoeologues, including many2many, one2many & one2one.

OUTPUT FORMAT and RETURN FIELDS

A tab-separated file with six columns. Each row of the output file describes a homology relation between a query gene and a target gene. Output contains the following columns:

  • target_id
      ID of target homologous gene (Ensembl gene_stable_id).

  • ref_org
      Name of the reference organism.

  • type
      Type of homology relationship.

  • query_id
      ID of the query gene.

  • ident_target
      %protein sequence identity with respect to target length.

  • ident_query
      %protein sequence identity with respect to query length.

HOMOLOGY CRITERIA

Ensembl Compara gene orthology and paralogy predictions are generated by a pipeline where maximum likelihood phylogenetic gene trees play a central role. They aim to represent the evolutionary history of gene families, i.e. genes that diverged from a common ancestor. These gene trees reconciled with their species tree have their internal nodes annotated to distinguish duplication or speciation events, and thus support the annotation of orthologous and paralagous genes, which can be part of complex one-to-many and many-to-many relations. [Adapted from: http://www.ensembl.org/info/genome/compara/homology_method.html].
