RSA-tools - Tutorials - retrieve-seq

In the left frame, under the title Sequence retrieval, click on the tool "retrieve sequence". A form appears, allowing you to select the parameters of sequence retrieval.

Retrieving sequences for a selected set of genes

    We will first retrieve upstream sequences for a set of 5 yeast genes involved in yeast phosphate metabolism.

    1. In the pop-up menu "Organism", select Saccharomyces cerevisiae

    2. In the text area under "Genes", type the following list of genes
      3. YBR093C
PHO8
pho11
PHO81
Pho84

    3. Inactivate the option Prevent overlap with upstream ORFs.

    4. Leave all other options unchanged and click on the button GO

    Remarks

    • Gene names must be separated by carriage returns, because only the first word of each line is considered as a query.
    • Notice that gene names are case-insensitive.
    • Genes can be specified either by the systematic ORF identifier or by a common name. Synonyms are supported.

    After a few seconds, the result appears, in the form of a link to the sequences, followed by a series of Buttons.

    Click on the link, to see the upstream sequence of the selected genes. Check the sizes of the sequences obtained. Click on the Back button to come back to the previous result page.

    Below the link, you can see a series of buttons, which will allow you to send the retrieved sequences to the next task (as we will see in the next tutorials). For the time being, do not click these buttons.

Preventing overlap with upstream ORFs

    1. Come back to the retrieve sequence form.

    2. Perform the same operations as above, but this time, activate the option Prevent overlap with upstream ORFs.

    3. Compare the size of the sequences with the previous result.

Interpretation of the results

    The default size for yeast upstream sequences is 800bp. The reason for this choice is that 99% of the regulatory elements found in TRANSFAC (the Transcription Factor Database) are located between -1 and -800, relative to the start codon.

    However, the median value for the distance between a gene and its upstream neighbour is 450 bp. Thus, with a distance of 800bp, most upstream regions would include a fragment of coding sequence from the upstream ORF, which might bias subsequent analyses.

    The situation is even worst in bacteria, due to the organization of genes into operons. In Escherichia coli, 25% of the genes have an upstream neighbour closer than 50bp. When this neighbour is on the same strand as the gene, it might indicate that they belong to the same operon.

    For most organisms, it is thus preferible to prevent overlap with upstream ORFs. A notable exception is the yeast Saccharomyces cerevisiae. We evaluated the results obtained when this option is activated and inactivated respectively, and it is clear that many patterns escape detection when upstream ORFs are clipped. One possible reason is that the annotations of Saccharomyces cerevisiae contain many false ORFs (some auhors estimate that there are ~1000 false ORFs in the original annotation).

    This is particularly well examplified with the yeast PHO genes : when the option is activated, PHO8 and PHO64 upstream regions are truncated at 180 and 237 base pairs, respectively. In both cases, the putative upstream ORF is very short (~300bp), and there is no experimental evidence for the existence of a gene. In addition, several cis-acting elements have been shown to be located in the regions corresponding to these questionable ORFs. When the sequences are clipped, the remaining Pho4p are not sufficiently over-rerpesented for allowing their detection in a pattern discovery programs.

    The PHO family is an extreme case, and when one disposes of a larger family, it is less likely for false ORFs to prevent detection of cis-acting elements. However, even in these cases, we noticed a loss in sensitivity when the option was activated.

    In summary, we propose the following strategy :

    • When working with bacteria, systematically activate the option prevent overlap with upstream ORFs, to prevent including too many coding sequences when the genes are located in operons.
    • With eukaryotes, inactivate this option, to circumvent the weaknesses of gene boundary annotations.

Retrieving sequences for all the genes of a genome

    As an illustration fo the retrieval of all sequences, we will now use the same program to retrieve sequences on the other side of the genes : downstream sequences.

    1. Open the retrieve sequence form by clicking retrieve sequence in the left frame.

    2. Make sure that the selected organism is Saccharomyces cerevisiae

    3. Besides the label Genes, select "all".

    4. Do not enter anything in the "Gene" box (it would be ignored anyway).

    5. Select "downstream" as Sequence type.

    6. In the From box, type "1".

    7. In the to box, type "50".

    8. Select "ORF id + Gene name" in the pop-up menu Sequence label

    9. Click GO

Remark about the reference position

    The reference position (coordinate 0) depends on the type of sequences to be retrieved :

    • For upstream sequences, the reference position is the ORF start, i.e. the first nucleotide of the start codon. Negative coordinates are used to indicate sequences located upstream the start codon, i.e. in the 5'end flank of the gene (e.g. from -1 to -800 for the yeast).
    • On the contrary, when selecting downstream sequences, the reference position (position 0) is the ORF end, i.e. the last nucleotide of the stop codon. Positive coordinates are used to represent sequences located downstream the stop codon, i.e. the 3'end flank of the gene.

Positive/negative coordinates

    Notice that selecting upstream sequences with positive coordinates amounts to pick coding sequences on the 5' side of the gene. As an exercise, try to extract all start codons for a given organism (say Saccharomyces cerevisiae). Check that the codons correspond to the expectation (ATG).

    Reciprocally, for downstream sequences, negative coordinates could be specified to retrieve coding sequences on the 3'end of the gene. As en exercise, try to extract all the stop codons for the same organism. Check that the stop codons correspond to the expectation (TAA, TAG or TGA).
