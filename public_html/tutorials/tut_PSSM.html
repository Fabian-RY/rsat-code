Introduction

    Position-specific scoring matrices (PSSM) offer a sensitive way to represent the specificity of transcription factor/DNA interfaces. PSSM are built on the basis of a set of known binding sites for the factor of interest.

    Knowledge-based PSSM can be obtained from various transcription factor databases (e.g. TRANSFAC).

    PSSM can be used to detect occurrences of a motif in sequences. Matrix-based pattern matching will be introduced in the tutorial on patser.

    Several pattern-discovery programs allow to detect significant motifs represented as PSSM from a set of unaligned sequences. These programs have been used to predict regulatory elements in the regulatory regions of sets of co-regulated genes. Matrix-based pattern discovery will be discussed in the tutorials on the gibbs sampler and consensus.

Example

    The matrix below was obtained from SCPD, the Saccharomyces cerevisiae promoter database.

    The PHO4 matrix has been built from an alignment of 12 binding sites for the yeast transcription factor Pho4p. 

    A |  3   2   0  12   0   0   0   0   1   3
C |  5   2  12   0  12   0   1   0   2   1
G |  3   7   0   0   0  12   0   7   5   4
T |  1   1   0   0   0   0  11   5   4   4

    Each row represents one residue (A, C, G or T), and each column a position in a set of aligned binding sites. Some positions are perfectly conserved across all known binding sites (the motif CACGT starting at the 3rd position), whereas some other positions present two choices (e.g. G or T at position 8), and other position can contain any letter, but with different frequencies (e..g first and last position).

    When the matrix isused to scan sequences for putative Pho4p binding sites, the more conserved positions qimpose stronger constraints than those where any nucleotide can be found. This allows thus a more quantitative evaluation of the binding affinity than string-based representations.

    One weakness of these matrices is that they do not take into account higher order dependencie, i.e. influence between a residue and the successive ones. Thus, it is not possible to specify "either CACGTGGG or CACGTTTT" with a PSSM : if one builds a matrix where G and T are allowed at the 3 last positions, any combination of them will be allowed (e.g. CACGTGTG, CACGTTGT). Higher order dependencies can be represented with more elaborated methods, such as Hidden Markov Models (HMM), which are ou of scope for this tutorial.

Converting and analyzing PSSM

    The RSAT include a program called convert-matrix, which allows to extract a PSSM from the output files of different programs (consensus, gibbs, MEME, clustal).

    I still did not find the time to write a tutorial for this program, but you can already read the manual, and do your own test with the demo data set (click DEMO at the bottom of the form).

    Matrix information

    We will import a matrix representing the binding specificity of the yeast transcription factor Abf1p, and analyze the different parameters which characterize this PSSM.

    1. Connect SCPD, the Saccharomyces cerevisiae promoter database. To access the list of sites by factor from the SCPD home page, click on the Section "Regulatory elements and transcriptional factors".
    2. Click on the link to ABF1, and then on the button Get matrix. The following PSSM is displayed. 
      >ABF17121SCPD

A   0   0  11   2   3   7   3   5   5  14   0   1
T  14   0   2   1   6   2   7   3   0   0   0   0
G   0   0   0   0   2   3   1   2   3   0   1  13
C   0  14   1  11   3   2   3   4   6   0  13   0.

      Some basic questions

      • What is the alphabet size ?
      • What is the matrix width ?
      • How many inding sites were used to build this matrix ?
      • How many Abf1p binding sites are currently stored in SCPD (click on the button Get sites)?

    3. We will now analyze the content of this matrix with the program convert-matrix. Copy the matrix from SCPD to the text area in the form.

    4. The current version of convert-matrix does not support the SCPD format (this will be fixed soon), but you can easily enter it in consensus format. For this, you need to perform 3 operations:
      • replace the header line (>ABF17121SCPD) by the words MATRIX 1. It should give the following result:
      • Add a pipe character after each residue (A, C, G, T).
      • Suppress the dot at the end of the matrix.
      The result should look like this: 
      MATRIX 1

A |   0   0  11   2   3   7   3   5   5  14   0   1
T |  14   0   2   1   6   2   7   3   0   0   0   0
G |   0   0   0   0   2   3   1   2   3   0   1  13
C |   0  14   1  11   3   2   3   4   6   0  13   0

    5. Select the following options proposed as return fields:
      • alignment (counts)
      • frequencies
      • weights
      • information
      • margins
      • parameters

    6. click GO.
