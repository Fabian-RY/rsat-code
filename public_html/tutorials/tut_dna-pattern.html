RSA-tools - Tutorials - dna-pattern

    DNA-pattern is a program that allow to match known patterns against a sequence set. The program has some features that make it particularly well suited for the recognition of regulatory motifs in DNA sequences, in short :

    1. Motifs can automatically be searched on both strands of the sequences.
    2. Motis can contain any letter from the degenerate nucleotide alphabet.
    3. It is possible to specify a number of allowed substitutions. The program will not allow deletions or insertions in the motif. This is important because many transcription factors allow some degeneracy, but insertions and deletions usually prevent the factor from recognizing a site.

    Let us suppose that you are searching the positions of putative GATA boxes and Hap sites within the upstream regions of the genes from the GDH family (GDH1, GDH2, and GDH3). We will first retrieve the sequences upstream from your genes.

    1. Retrieve upstream sequences from -800 to -1 for the following yeast genes (as you have seen in the tutorial on sequence retrieval) 
      GDH1
GDH2
GDH3

      Now we have the upstream sequences, we will scan them with the consensi for GATA boxes and HAP sites. At the top of the result page, a series of buttons are presented. These buttons allow you to send your sequence to a selection of sequence analysis programs. Click on dna-pattern (IUPAC). A new form appears.

      Note that the search will automatically be performed on the sequences you just retrieved (sequence transferred from your previous query). This differs from the form you would receive by clicking on "dna-pattern" in the left panel, and which would contain an additional box for entering your own sequences.

    2. In the Query pattern(s) box, we will enter the patterns to search for. Eah pattern must come on a separate line. The first word of each line is the string to search for, the second word is an identifier for this pattern. Type the following text in the Query pattern(s) box:
      3. GATWA        Gata_box
CCAAY        Hap_site
      Note the use of degenerate IUPAC degenerate code: the W from GATWA on the first line means "either A or T", the Y from CCAAY on the second line means "either C or T".

    3.  click GO.

      4. You see now the positions of all matches with the patterns you entered within the upstream sequence of GDH1, GDH2 and GDH3. Each line shows a single match, and the different columns indicate respectively:

      • pattern identifier
      • strand on which the match was found (D for direct, R for Reverse)
      • pattern searched for (i.e. the consensus you provided)
      • sequence in which it was found
      • starting position of the match
      • end position of the match
      • matching sequence. The matching bases are indicated in UPPERCASES. The 4 flanking bases at left and right are in lowercases.
      • matching score. In this case all scores equal 1, but we will see later how to use this column.
      Notice that positions are returned in negative coordinates, relative to the end of the sequence (the last nucleotide has position -1). This option (Origin=end) is particularly useful for analyzing regulatory sequences, but it can be inactivated in other cases.

      We will now display the same results graphically.

      1. Click on the Feature map button on the top of the result page.

      2. In the Title box, type
        3. Gata boxes and Hap sites in the upstream regions from GDH genes

      3. after the title Display limits, fill

        • the from box with -800,

        • the to box with 0

      4. In the pop-up menu "feature handle", select symbol

      5. make sure the Dynamic map option is checked.

      6. Leave other parameters unchanged and click GO.

      After a few seconds, the feature map should appear. A few comments:

      • Gata boxes appear in blue, Hap sites in red

      • Color boxes are either above or below the horizontal black lines, accordingly to the strand (Watson or Crick respectively) of the match.

      • A specific symbol is associated to each pattern, allowing to distinguish them when the feature map is printed in black and white

      • Coordinates are provided with reference to the ORF starting position, negative values indicate an upstream, and positive values for the coding sequences (0 corresponds to the first nucelotide ot the start codon).

      • Just for the fun, note the nice repetition of Hap-Gata site pairs in the upstream sequence of GDH1.

      • If your browser is recent, the map is dynamic. With your mouse, position the cursor just above one of the sites in the sequences. Look now at the bottom border of your Netscape window. The complete information about this site is displayed. Move the cursor to another site and check that the information is well updated. If you are using Internet Explorer, make sure to activate the status bar (in the View menu).

      You can now come back to the tutorial main page and try the next tutorial.
